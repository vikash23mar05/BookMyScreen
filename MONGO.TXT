

MongoDB standalone does support transactions — but only limited (single-document).

🧠 Why this limitation exists
🔹 1. Transactions need a replication mechanism

MongoDB transactions rely on:

oplog (operations log)
replication coordination
rollback + recovery guarantees

These come from 👉 Replica Set architecture

In standalone MongoDB:

❌ No oplog
❌ No replication
❌ No rollback coordination

👉 So MongoDB cannot guarantee ACID across multiple documents


🚀 Why Replica Set fixes it

Replica set has:

Primary + Secondary nodes
Oplog (history of operations)
Rollback capability

So MongoDB can:

Start transaction
Track changes
Commit OR rollback safely
